# 🔍 SearchXNG Telegram Bot

**SearchXNG Telegram Bot** — это высокопроизводительный Telegram-бот для поиска информации в интернете в реальном времени с использованием больших языковых моделей (LLM) через любой OpenAI-совместимый API (локальный инференс Ollama, облачные провайдеры OpenRouter, OpenAI, DeepInfra и др.) и поискового движка SearXNG.

Бот поддерживает работу в обычном режиме чата и в **инлайн-режиме (inline mode)** для мгновенного получения ответов прямо в поле ввода любого чата Telegram.

---

## 🌟 Ключевые преимущества: Точность и Актуальность

* **🎯 Высочайшая точность и достоверность**: Бот не просто возвращает первые попавшиеся ссылки. Он сопоставляет информацию из различных независимых источников, отсеивает противоречивую информацию и делает упор на перекрестную проверку (cross-verification) фактов.
* **📅 Акцент на актуальность данных (Контекст 2026 года)**: В системные инструкции встроен строгий контекст текущего времени (2026 год). Бот умеет распознавать устаревшие статьи, не выдает прошлые версии продуктов за «последние на сегодняшний день» и явно указывает, если в источниках отсутствуют свежие данные по теме.
* **⚡ Оптимизированный инлайн-режим**: В инлайн-режиме бот сохраняет структуру абзацев и списков, но автоматически делает ответы максимально ёмкими и сжатыми (в пределах 80-100 слов), убирая длинные подробные трактаты и заголовки `###`.

## 🖼️ Схема работы (Жизненный цикл запроса)

Поисковая система бота построена по принципу многошагового автономного агента (Deep Research Agent), который выполняет глубокий многоуровневый сбор информации и перекрестную валидацию данных.

```mermaid
graph TD
    %% Ноды начала и конца
    User([Пользователь Telegram]) :::startEnd -->|Ввод: Текст / Картинки / Файлы| TG[Telegram Bot API]
    
    %% Этап 0: Входной анализ
    TG --> CheckInput{Тип входа?}
    CheckInput -->|Картинка| Vision[Vision-анализ: Описание картинки]:::accent
    CheckInput -->|Файл PDF/Docx/Xlsx/xls| ExtractDoc[Парсинг документов & Краткая выжимка]:::accent
    CheckInput -->|Текст| InitSession[Инициализация сессии SQLite]
    
    Vision -->|Описание добавлено к контексту| InitSession
    ExtractDoc -->|Конспект добавлен к контексту| InitSession
    
    InitSession --> Classify[0. Авто-классификация запроса: FACT / COMPREHENSIVE]
    Classify --> CheckMode{Режим поиска?}
    
    %% ВЕТКА FAST MODE
    CheckMode -->|Fast Mode| FastInit[Первичный сбор: Генерация 3 запросов без рассуждений think]
    FastInit --> FastSearchLoop[Параллельный сбор данных SearXNG]
    FastSearchLoop --> FastConcat[Объединение сырых материалов сайтов]
    FastConcat --> FastSynth[Синтез финального ответа без think]
    
    %% ВЕТКА DEEP MODE
    CheckMode -->|Deep Mode| DeepLoop[Итеративный цикл сбора данных: до 3 попыток]:::accent
    
    %% Фаза 1.1: Первичный сбор
    DeepLoop --> Phase11[1.1. Первичный сбор: Генерация 3 запросов с think-рассуждениями]
    Phase11 --> Scrap11[Параллельный опрос SearXNG]
    Scrap11 --> CheckSites11{Доступно >= 3 сайтов?}
    
    %% Цикл обхода блокировок (Refining Loop)
    CheckSites11 -->|Нет| RefineQuery11[Генерация альтернативного запроса с обходом сбоящих URL]:::warning
    RefineQuery11 --> Scrap11
    
    CheckSites11 -->|Да| AINav11[ИИ-навигатор: Выбор до 5 лучших ссылок по сниппетам]
    AINav11 --> Fetch11[Полное скачивание страниц в параллельном режиме]
    Fetch11 --> Sum11[Конспектирование: Постраничная суммаризация для каждого запроса]
    
    %% Фаза 1.2: Уточняющий сбор
    Sum11 --> Phase12[1.2. Уточняющий сбор: Поиск пробелов в логе исследования]
    Phase12 --> Scrap12[Параллельный опрос SearXNG]
    Scrap12 --> CheckSites12{Доступно >= 3 сайтов?}
    CheckSites12 -->|Нет| RefineQuery12[Генерация альтернативного запроса]:::warning
    RefineQuery12 --> Scrap12
    CheckSites12 -->|Да| AINav12[ИИ-навигатор: Выбор ссылок]
    AINav12 --> Fetch12[Скачивание & Постраничная суммаризация]
    
    %% Фаза 2.1: Сбор мнений
    Fetch12 --> Phase21[2.1. Сбор мнений: Поиск отзывов, опыта и обсуждений на форумах]
    Phase21 --> Scrap21[Параллельный опрос SearXNG]
    Scrap21 --> CheckSites21{Доступно >= 3 сайтов?}
    CheckSites21 -->|Нет| RefineQuery21[Генерация альтернативного запроса]:::warning
    RefineQuery21 --> Scrap21
    CheckSites21 -->|Да| AINav21[ИИ-навигатор: Выбор ссылок]
    AINav21 --> Fetch21[Скачивание & Постраничная суммаризация мнений]
    
    %% Фаза 2.2: Перекрестный сбор
    Fetch21 --> Phase22[2.2. Перекрестный сбор: Поиск противоречий и сомнительных фактов]
    Phase22 --> Scrap22[Параллельный опрос SearXNG с фильтром по времени month]
    Scrap22 --> CheckSites22{Доступно >= 3 сайтов?}
    CheckSites22 -->|Нет| RefineQuery22[Генерация альтернативного запроса]:::warning
    RefineQuery22 --> Scrap22
    CheckSites22 -->|Да| AINav22[ИИ-навигатор: Выбор ссылок]
    AINav22 --> Fetch22[Скачивание & Постраничная суммаризация противоречий]
    
    %% Перекрестный анализ и валидация
    Fetch22 --> CrossVerification[Перекрестная верификация: Отчет по фактам и мнениям]
    CrossVerification --> FreshnessCheck{Проверка актуальности: Данные за 2026 свежие?}
    
    %% Временной рекурсивный цикл
    FreshnessCheck -->|Нет: Данные устарели / Неполные| LoopBack[Запуск повторной итерации с новыми временными маркерами]:::warning
    LoopBack --> DeepLoop
    
    %% Выход из цикла
    FreshnessCheck -->|Да / Лимит попыток| ThinkStage[Генерация итогового ответа с каскадом LLM и think]
    
    %% Объединение к выходу
    FastSynth --> SaveHistory[Сохранение истории диалога и сессии в SQLite]
    ThinkStage --> SaveHistory
    
    SaveHistory --> Out([Итоговый структурированный ответ с источниками и временем]) :::startEnd

    %% Стилизация схемы
    classDef default fill:#ffffff,stroke:#333333,stroke-width:2px,color:#000000;
    classDef accent fill:#f5f5f5,stroke:#000000,stroke-width:2px,stroke-dasharray: 5 5,color:#000000;

    class User,Out startEnd;
    class Vision,ExtractDoc,DeepLoop accent;
    class RefineQuery11,RefineQuery12,RefineQuery21,RefineQuery22,LoopBack warning;
```

## ⚡ Инлайн-режим (Inline Mode)

Бот поддерживает вызов из любого чата Telegram, группы или приватного диалога без необходимости переходить в чат с самим ботом.

* **Как использовать**:
  1. В поле ввода любого чата начните писать: `@username_бота <ваш поисковый запрос>` (например, `@SearchXNG_Bot как вырастить авокадо`).
  2. Бот выполнит фоновый поиск и покажет превью (`⚡ Быстрый поиск` и `🔍 Глубокий поиск`) над полем ввода.
  3. Кликните по результату — бот отправит отформатированный структурированный ответ в чат от вашего имени.
  
* **Как это устроено технически**:
  - При отправке inline-запроса бот перехватывает его через обработчик `inline_handler`.
  - Запускается поисковая сессия с флагом `is_inline=True`.
  - В системный промпт автоматически подмешивается инструкция на лаконичность (лимит 80-100 слов, запрет на заголовки `###`), благодаря чему ответ сохраняет читаемую структуру абзацев и списков, но занимает минимум места на экране.
  - Результат передается обратно в Telegram в виде объекта `InlineQueryResultArticle`.

## 🧠 Подробный разбор алгоритмов работы

Обработка поисковых запросов делится на два режима, каждый из которых работает по собственному конвейеру (pipeline):

### 1. ⚡ Быстрый режим (Fast Mode)
Предназначен для получения моментальных и фактологически точных ответов на простые запросы.
* **Конвейер шагов**:
  1. **Входной Vision/Документ-анализ**: Если присланы файлы или фото, извлекается текстовое описание (визионером или парсером) и добавляется в контекст.
  2. **Авто-классификация**: Быстрая проверка запроса через LLM на принадлежность к типу `FACT` или `COMPREHENSIVE`. Для простых фактов лимит сбора сайтов увеличивается до 10 для сбора максимального количества точечных источников.
  3. **Первичный сбор (Генерация 3 запросов)**: Модель генерирует 3 поисковых запроса без использования тегов `<think>` для высокой скорости ответа.
  4. **Параллельный поиск & Скачивание**: Поиск в SearXNG по 3 запросам одновременно. Все найденные ссылки проходят быструю проверку (`quick_check_url`) на HTTP-статус и тип содержимого (пропуск бинарников).
  5. **Синтез без рассуждений**: Текст скачанных сайтов объединяется в единый контекст. Финальный ответ генерируется напрямую без монолога `<think>` для экономии времени (ответ формируется за 15–30 секунд).

### 2. 🔍 Глубокий режим (Deep Mode)
Многоэтапный агентный цикл исследования с перекрестным анализом, борьбой с блокировками и валидацией данных.
* **Детальный разбор фаз**:
  - **Входная обработка**: Описание картинок через мультимодальную модель, извлечение текстов из `PDF`, `Docx`, `Xlsx` с помощью библиотек (`pypdf`, `python-docx`, `openpyxl`).
  - **Классификация**: Определение глубины запроса.
  - **Фаза 1.1. Первичный сбор**:
    - Модель генерирует 3 поисковых запроса на основе подробных рассуждений в `<think>`.
    - **Поисковый цикл**: Для каждого запроса запускается поиск SearXNG. Все сайты-кандидаты проходят быструю асинхронную проверку доступности.
    - **Обход блокировок (Refining Loop)**: Если доступно менее 3 сайтов (блокировки Cloudflare, DDoS-защита, неработающие домены), ИИ автоматически переформулирует поисковый запрос, учитывая список сбоивших URL, и запускает поиск заново (до 3 попыток на запрос).
    - **ИИ-Навигатор ссылок**: Для прошедших проверку сайтов вызывается LLM, которая на основе сниппетов и заголовков выбирает до 5 наиболее релевантных и авторитетных индексов.
    - **Скачивание и постраничная суммаризация**: Выбранные страницы скачиваются, очищаются от HTML-мусора, меню и рекламы (BeautifulSoup) и индивидуально суммаризируются через ИИ-модель с записью конспектов в физический лог исследования (`data/research_log_{chat_id}.txt`).
  - **Фаза 1.2. Уточняющий сбор**: ИИ анализирует накопленный лог исследования, находит информационные пробелы по теме и формирует 3 новых точечных запроса для детализации данных. Сбор, навигация и конспектирование повторяются.
  - **Фаза 2.1. Сбор мнений и опыта**: Специальная генерация запросов, нацеленных на поиск живого опыта людей, отзывов на форумах, Reddit, GitHub, баг-трекерах и в сообществах.
  - **Фаза 2.2. Перекрестный сбор (Противодействие дезинформации)**: ИИ целенаправленно ищет противоречия и сомнительные факты, ограничивая время выдачи (параметр `time_range="month"` в SearXNG).
  - **Перекрестный анализ (Cross-Verification)**: Модель сопоставляет два блока данных (справочные факты из Фаз 1.1/1.2 и практический опыт/отзывы из Фаз 2.1/2.2). Формируется аналитический отчет: выявление противоречий между теорией и практикой, классификация информации на доказанные факты и субъективные мнения.
  - **Контроль свежести 2026**: Модель-валидатор проверяет, содержит ли отчет свежие данные. Если обнаружены устаревшие версии или пробелы, запускается рекурсивная итерация сбора с жесткими временными фильтрами (до 3 полных циклов).
  - **Итоговый монолог `<think>` и каскад LLM**:
    - Модель формулирует финальный ответ с подробным внутренним рассуждением в теге `<think>`.
    - **Каскадный обход сбоев**: Если выбранная пользователем модель не отвечает или возвращает ошибку, бот автоматически пробует стартовую модель 3 раза с интервалом в 2 секунды. При полной недоступности бот запускает каскадный опрос резервных моделей из API (`PREFERRED_BACKUP_MODELS`) с задержкой 5 секунд, гарантируя получение ответа даже при перегрузке основного API-провайдера.
  - **Вывод**: Ответ с пулом использованных источников, временем выполнения и возможностью сохранить сессию в SQLite архив.

## 🛠️ Стек технологий

* **Язык программирования**: Python 3.11+
* **Фреймворк для Telegram**: Aiogram 2.x
* **Поисковая база**: SearXNG (приватный метапоисковик)
* **Инференс LLM**: Любой OpenAI-совместимый API (локальный Ollama, облачные провайдеры OpenRouter, OpenAI, DeepInfra, Mistral AI и др.)
* **База данных**: SQLite (сессии, кэш мультимодальности и история диалогов)
* **Оркестрация**: Docker & Docker Compose

## 🚀 Быстрый старт

### 1. Подготовка окружения
Склонируйте репозиторий и создайте файл конфигурации `.env` на основе примера:
```bash
cp .env.example .env
```

Отредактируйте `.env` и укажите ваши настройки (см. раздел [Настройка переменных окружения](#-настройка-переменных-окружения)).

### 2. Запуск через Docker Compose

Для развертывания бота в Docker-контейнере со всеми зависимостями выполните:

```bash
# Сборка и запуск контейнеров в фоновом режиме
docker compose -f docker-compose.sum.yml up -d --build
```

## 📝 Настройка переменных окружения (`.env`)

В файле `.env` настраиваются параметры подключения к LLM-провайдеру и поисковику:

| Переменная | Описание | Пример значения |
| :--- | :--- | :--- |
| `TELEGRAM_BOT_TOKEN` | Токен вашего Telegram-бота, полученный от @BotFather | `1234567890:ABCdef...` |
| `OLLAMA_BASE_URL` | Базовый URL-адрес OpenAI-совместимого API-эндпоинта | `http://ollama:11434` (Ollama), `https://openrouter.ai/api` (OpenRouter), `https://api.openai.com` (OpenAI) |
| `OLLAMA_MODEL` | Название модели ИИ для генерации ответов и поиска | `llama3` / `gemma2` (Ollama), `meta-llama/llama-3-70b-instruct` (OpenRouter), `gpt-4o` (OpenAI) |
| `OLLAMA_API_KEY` | Токен авторизации / API-ключ выбранного провайдера | `sk-or-v1-...` (OpenRouter API Key), `sk-proj-...` (OpenAI API Key) или оставьте пустым для локального Ollama |
| `SEARXNG_URL` | URL-адрес вашего инстанса поисковика SearXNG | `http://searxng:8080` |
